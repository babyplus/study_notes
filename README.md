# 抽象工厂  

## 意图  

提供一个接口以创建一系列相关或相互依赖的对象，而无须指定它们的具体的类。  

## 例子

[abstract_factory](abstract_factory/samples)

# 适配器  

## 意图  

将一个类的接口转换成客户希望的另外一个接口，使得原本由于接口不兼容而不能一起工作的那些类可以一起工作。

## 例子

[adapter](adapter/samples)

# 桥接  

## 意图  

将抽象部分与它的实现部分分离，使它们可以独立地变化。  

## 例子

[bridge](bridge/samples)

# 生成器  

## 意图  

将一个复杂对象的构建与它的表示分离，使得同样的构建过程可以创建不同的表示。  

## 例子

[builder](builder/samples)

# 职责链  

## 意图  

使多个对象都有机会处理请求，从而避免请求的发送者和接收者之间的耦合关系。将这些对象连成一条链，并沿着这条链传递该请求，直到有一个对象处理它为止。

## 例子

[chain_of_responsibility](chain_of_responsibility/samples)

# 命令  

# 意图  

将一个请求封装为一个对象，从而使你可以用不同的请求对客户进行参数化，对请求排队或记录请求日志，以及支持可撤销的操作。  

## 例子

[command](command/samples)

# 组合  

## 意图  

将对象组合成树形结构以表示"部分-整体"的层次结构。组合使得用户对单个对象和组合对象的使用具有一致性。

## 例子

[composite](composite/samples)

# 装饰模式  

## 意图  

动态地给一个对象添加一些额外的职责。就增加功能来说，装饰模式比生成子类更为灵活。

## 例子

[decorator](decorator/samples)

# 外观  

## 意图  

为子系统中的一组接口提供一个一致的界面，Facade模式定义了一个高层接口，这个接口使得这一子系统更加容易使用。  

## 例子

[facade](facade/samples)

# 工厂方法  

## 意图  

定义一个用于创建对象的接口，让子类决定实例化哪一个类。工厂方法使一个类的实例化延迟到其子类。

## 例子

[factory_method](factory_method/samples)

# 享元  

## 意图  

运用共享技术有效地支持大量细粒度的对象。

## 例子

[flyweight](flyweight/samples)

# 解释器  

## 意图  

给定一个语言，定义它的文法的一种表示，并定义一个解释器，这个解释器使用该表示来解释语言中的句子。

## 例子

[interpreter](interpreter/samples)

# 迭代器  

## 意图  

提供一种方法顺序访问一个聚合对象中的各个元素，而又不需要暴露该对象的内部表示。  

## 例子

[iterator](iterator/samples)

# 中介者    

# 意图  

用一个中介对象来封装一系列的对象交互。中介者使各对象不需要显式地相互引用，从而使其耦合松散，而且可以独立地改变它们之间的交互。  

## 例子

[mediator](mediator/samples)

# 备忘录  

## 意图  

在不破坏封装的情况下，捕获一个对象的内部状态，并在该对象之外保存这个状态。这样以后就可将该对象恢复到原先保存的状态。  

## 例子

[memento](memento/samples)

# 观察者  

## 意图  

定义对象间的一种一对多的依赖关系，当一个对象的状态发生改变时，所有依赖于它的对象都得到通知并被自动更新。  

## 例子

[observer](observer/samples)

# 原型  

## 意图  

用原型实例指定创建对象的种类，并且通过拷贝这些原型创建新的对象。

## 例子

[prototype](prototype/samples)

# 代理  

## 意图  

为其他对象提供一种代理以控制对这个对象的访问。  

## 例子

[proxy](proxy/samples)

# 单件  

## 意图  

保证一个类仅有一个实例，并提供一个访问它的全局访问点。

## 例子

[singleton](singleton/samples)

# 状态  

## 意图  

允许一个对象在其内部状态改变时改变它的行为。对象看起来似乎修改了它的类。

## 例子

[state](state/samples)

# 策略  

## 意图    

定义一系列算法，把它们一个个封装起来，并且使它们可互相替换。该模式使得算法可以独立于客户程序而变化。本模式使得算法可独立于使用它的客户而变化。

## 例子

[strategy](strategy/samples)

# 模板方法  

## 动机  

定义一个操作中算法的骨架，将一些步骤延迟到子类中。子类不改变一个算法的结构即可重定义算法的某些特定步骤。  

## 例子

[template_method](template_method/samples)

# 访问者  

## 意图  

表示一个作用于某个对象结构中的各元素的操作。它使你可以在不改变各元素的类的前提下定义作用于这些元素的新操作。  

## 例子

[visitor](visitor/samples)
